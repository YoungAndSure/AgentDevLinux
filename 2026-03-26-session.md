参与 Linux 内核开发的步骤：

## 1. 找到合适的贡献点

- **驱动分支**：Linux 内核很大，新人多从驱动开发入手，如 staging、IIO、USB、GPU 等子系统
- **Documentation 和代码清理**：相对容易入门
- **Bug fix**：在内核邮件列表或 GitHub mirror 上查找 reported bugs

推荐资源：
- [kernel.org/doc](https://www.kernel.org/doc/html/latest/) — 内核文档
- [kernelnewbies.org](https://kernelnewbies.org/) — 适合新人的入门指南
- [lore.kernel.org](https://lore.kernel.org/) — 内核邮件列表归档

## 2. 理解问题和代码

- 用 `git log`、`git blame` 追踪代码历史
- 阅读相关子系统的 Mailing List 讨论
- 理解 `CONFIG_` 选项和 Kconfig 依赖关系

## 3. 提交 CR

Linux 内核开发使用邮件列表而非 GitHub PR：
```bash
git format-patch -1  # 生成 patch
git send-email --to <maintainer@example.com> patch
```

## 建议

1. 先从小型 fix 开始（typo、coding style、简单 bug）
2. 阅读 `Documentation/process/` 了解提交流程
3. 找到你想参与子系统的 maintainer 和 mailing list

**原理**：FUSE 是 Linux 内核模块，允许**用户态程序**实现文件系统


**本质**：镜像根本没有挂载到宿主机，宿主机只是读取/写入镜像文件
